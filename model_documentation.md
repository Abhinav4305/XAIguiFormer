# XAIguiFormer Model Documentation

## 1. Overview and Core Objective
The **XAIguiFormer** (Explainable Artificial Intelligence Guided Transformer) is a deep learning architecture designed for brain disorder identification using EEG-based connectomes. 

Its primary objective is twofold:
1. **Accurate Classification**: Utilize structural graphs (connectomes) extracted from EEG data to classify brain disorders (e.g., normal vs. abnormal brain activity).
2. **XAI-Guided Enhancement**: Instead of just using Explainable AI (XAI) passively to interpret the trained model, XAIguiFormer actively integrates the XAI module during the forward pass. It computes the internal contributions of frequency bands and injects this explanation back into the self-attention mechanism, refining the transformer to focus on the most relevant features.

## 2. Architecture and Core Components

The model ([models/XAIguiFormer.py](file:///Users/abhinavbhargava/Downloads/XAIguiFormer/models/XAIguiFormer.py)) is structured as a pipeline with three primary components:

### A. Connectome Encoder ([modules/connectome_encoder.py](file:///Users/abhinavbhargava/Downloads/XAIguiFormer/modules/connectome_encoder.py))
- **Purpose**: Processes the raw graph data (nodes representing EEG channels, edges representing connectivity).
- **Mechanism**: 
  - Passes the input node and edge features through initial Linear embeddings.
  - Applies a stack of Graph Neural Network (GNN) layers. By default, it uses **GINEConv** to capture the topological structure of the connectome.
  - Applies a pooling operation (e.g., `mean` pooling) across nodes to generate a single aggregate representation vector for each distinct frequency band graph.

### B. XAIguiTransformer Encoder ([modules/transformer.py](file:///Users/abhinavbhargava/Downloads/XAIguiFormer/modules/transformer.py))
- **Purpose**: Processes the sequence of frequency band representations generated by the Connectome Encoder. 
- **Mechanism**:
  - Treats each frequency band's representation as a token in a sequence.
  - Utilizes a custom **XAIguiAttention** mechanism. 
  - **Position Encoding**: Instead of standard absolute positional encoding, it uses a 2D Rotary Position Encoding (`dRoFE`) which incorporates the specific frequency band ranges as well as **demographic information** (like age or gender), anchoring the attention mechanism to biological traits.
  - An `AvgPool` and MLP classification head sit at the end of the transformer sequence to produce the logit predictions.

### C. Explainer Module ([modules/explainer.py](file:///Users/abhinavbhargava/Downloads/XAIguiFormer/modules/explainer.py))
- **Purpose**: Generates dynamic feedback to guide the transformer.
- **Mechanism**:
  - Uses an attribution method (default: **DeepLift**) to compute the contribution of each frequency band representation toward the initial prediction of the Transformer Encoder.
  - **The "Guide" Process**: The computed explanations are fed into the [XAIguiAttention](file:///Users/abhinavbhargava/Downloads/XAIguiFormer/modules/transformer.py#9-45) layer. The query (`q`) and key (`k`) projections are explicitly calculated from the *explanation* tensor rather than just the raw feature tensor, effectively forcing the attention map to weight relationships based on their actual significance to the predictive task.

## 3. Forward Pass Workflow
1. Raw graph data is embedded by the [ConnectomeEncoder](file:///Users/abhinavbhargava/Downloads/XAIguiFormer/modules/connectome_encoder.py#7-54).
2. The [XAIguiTransformerEncoder](file:///Users/abhinavbhargava/Downloads/XAIguiFormer/modules/transformer.py#94-134) makes an initial, baseline forward pass to generate a preliminary prediction `pred_y`.
3. The `Explainer` uses the input features and `pred_y` to generate an `explanation` (contributions of frequency bands).
4. The features are passed through the [XAIguiTransformerEncoder](file:///Users/abhinavbhargava/Downloads/XAIguiFormer/modules/transformer.py#94-134) a *second time*, this time being conditioned on the `explanation` to refine the self-attention matrices.
5. The final enhanced prediction is returned.

## 4. Hyperparameters and Specifications
Based on the configuration files (e.g., [TUAB_model.yaml](file:///Users/abhinavbhargava/Downloads/XAIguiFormer/configs/TUAB_model.yaml), [BEED_model.yaml](file:///Users/abhinavbhargava/Downloads/XAIguiFormer/configs/BEED_model.yaml)), typical parameters include:

### Graph Dimensions
- **Node Features (`num_node_feat`)**: Typically 16 or 19 (Flattened coherence matrix diagonals or dataset-specific EEG features).
- **Edge Features (`num_edge_feat`)**: 1 (Usually representing pseudo-wPLI edge weights / distance).
- **Hidden Features (`dim_node_feat` / `dim_edge_feat`)**: Configurable, usually `64` or `128`.

### Component Sizing
- **GNN Layers (`num_gnn_layer`)**: Range from 2 to 4 layers. GNN Type is natively `GINEConv`.
- **Transformer Layers (`num_transformer_layer`)**: Highly dependent on dataset size; `4` layers for smaller datasets like BEED, up to `12` layers for large datasets like TUAB.
- **Attention Heads (`num_head`)**: Typically `4` heads.
- **MLP Ratio (`mlp_ratio`)**: `4.0` (Determines the hidden dimension of the feed-forward network inside the transformer).

### Training Parameters
- **Dropout & Attn Drop**: `0.1`
- **Criterion**: Cross Entropy with optional Label Smoothing (e.g., `smoothing: 0.1`)
- **Optimizer**: AdamW with learning rates around `1e-4` to `4e-5`, weight decay of `0.01` or `1e-4`. 
- **Explainer Type**: `DeepLift` (Captum library implementation).

## 5. Extensibility
The architecture cleanly decouples the GNN spatial processing from the Temporal/Frequency Transformer processing. 
- You can swap the type of GNN in `gnn_type` (e.g., `GCNConv`, `GATConv`).
- You can swap the `explainer_type` to inject different XAI methods (e.g., `IntegratedGradients` instead of `DeepLift`).
